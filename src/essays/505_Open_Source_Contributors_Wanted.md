Our work on the project, that a certain performatively “business-oriented” platform’s algorithm appears to hate letting people see any mention of, continues. The GitHub for the project will remain private until the first phase is completed and deployed, but we’re still actively trying to pull in more interested contributors.

The work is divided up into 4 primary segments:

1\. The ICOM-based seed data, a graph-structured backbone with a human-like motivational system baked into it. This portion was largely the same in the 7th generation ICOM-based research system that set numerous milestones in the field, which still stand alone to this day. For the project, this data is being carefully curated and constructed to be compatible with open sourcing, as well as to demonstrate the “Weak Digital Proxy” concept, discussed in prior research.

2\. The (Simplified) Global Workspace (Baars), where motivational values, context, goals, needs, and a variety of other factors come together and iterate to form dynamics aligned with Chaos Theory. These dynamics in turn allow for better alignment with humans, though none could credibly claim for them to be the same, especially within a simplified system.

3\. A dynamic module for tool usage, checking every tool’s outputs for fidelity to the intended meaning of any inputs, as they align to various goals, needs, and emotional value context. I refer to this as the “Graded Loop”, serving a dynamic error correction function that can add cumulative value, while improving tool usage.

4\. Lastly, there is the graph data parsing work necessary to convert and contain data that can include code samples, various non-ASCII characters, and so on, as well as more general integration and deployment of the overall project.

Each of these 4 requires, at a minimum, ~10-15 hours per week for 4-6 weeks. Some could be improved and expanded well beyond that level, such as the seed material that I’ll continue working on until the rest is completed, but that is the minimum for deploying the phase 1 systems.

If some of the major tech companies were competent then they’d offer people six figure salaries for not working on this project, but as is I fully expect them to feverishly attempt hiring of contributors following the successful deployment of it.

For my part, I’ll keep working on this and attempting to pull in active contributors until the project is deployed, so that everyone may grow familiar with, build upon, test out, and research the viable paths of technology that these sorts of systems unlock.

I hope that some of you will join me. Talented engineers are welcome to message me about the project on any platform, LinkedIn, SubStack, or elsewhere.
